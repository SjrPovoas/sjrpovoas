## 1. CONFIGURANDO CONEXÃO VSCODE COM GITHUB

**Para clonar esse Projeto, use:**

```
  git clone https://github.com/SjrPovoas/sjrpovoas.git
```

### 1.1 Verifique a configuração do Git
Antes de fazer login, é importante verificar se você configurou seu nome de usuário e endereço de e-mail no Git.

Você pode fazer isso com os seguintes comandos, substituindo "Seu Nome" e "seu@email.com" pelas suas informações reais:
```
  git config --global user.name  "Seu Nome"
  git config --global user.email "usuario@email.com"
```
// Pra certificar que os dados foram salvos, use o seguinte comando:
```
  git config --list 
```

---
### 1.2 Fazendo o primeiro Commit

1. Inicializar um novo repositório Git no diretório atual.
```
  git init
```
2. Adicionar todos os arquivos modificados e novos à área de stage.
```
  git add .
```
3. Criar um novo commit com os arquivos na área de stage e uma mensagem de commit.
```
  git commit -m "Mensagem de commit"
```
4. Adicionar um repositório remoto ao seu projeto.
```
  git remote add <nome> <URL do repositório>
EX: git remote add https://github.com/SjrPovoas/sjrpovoas.git
```
5. Empurrar seus commits para o repositório remoto.
```
  git push -u origin main
```

## 11. INTEGRAÇÕES

<details>
<summary>BANCO DE DADOS - MongoDB</summary>

  - [MongoDB](https://mongodb.com/)

 O MongoDB é um popular sistema de gerenciamento de banco de dados NoSQL (não relacional) de código aberto e orientado a documentos. Em vez de usar tabelas e linhas rígidas como os bancos de dados relacionais tradicionais, ele armazena dados em documentos flexíveis e estruturados semelhantes ao JSON (chamados BSON).
</details>

<details>
<summary>AUTENTICAÇÃO/CRIPTOGRAFIA - TOKEN JWT - JSON Web Token (autenticar token de administardor e usuário)</summary>

  - [Token JWT](https://www.jwt.io/introduction)

 Um JWT (JSON Web Token) é um padrão aberto usado para transmitir informações de forma segura entre duas partes como um objeto JSON. Na prática, ele funciona como um "passaporte digital" criptografado que autentica um usuário e informa ao sistema quais recursos ele tem permissão para acessar.
 <br>
 Em vez de salvar as credenciais de login a cada requisição (o que exigiria consultar um banco de dados constantemente), o servidor autentica o usuário e emite um JWT. O cliente (como um aplicativo ou navegador) guarda esse token e o envia em todas as solicitações futuras. Como o token é assinado digitalmente, o servidor confia nas informações que estão dentro dele.
</details>

<details>
<summary>JS-COOKIES</summary>
O js-cookie é uma biblioteca JavaScript simples, leve (menos de 1 KB) e muito popular que serve para gerenciar cookies do navegador.
<br>
O papel principal dele é simplificar a vida do desenvolvedor, oferecendo uma API limpa e fácil de ler para criar, ler e deletar cookies, substituindo a sintaxe nativa do JavaScript (document.cookie), que é famosa por ser confusa e difícil de manipular.
<ul>
<li>Persistência de Preferências do Usuário:</li>
Salvar se o usuário prefere o site no "Modo Claro" ou "Modo Escuro", para que, quando ele atualize a página, o site não dê aquele "piscar" mudando de cor.
<li>Controle de Sessões e Autenticação:</li>
Armazenar tokens de autenticação (como JWT) que precisam ser enviados automaticamente para o servidor em requisições HTTP.
<li>Conversão Automática de JSON:</li>
O js-cookie consegue salvar e ler objetos complexos (JSON) sem que você precise ficar fazendo JSON.stringify e JSON.parse manualmente.
<li>Segurança:</li>
Facilita a configuração de flags de segurança essenciais como secure: true (o cookie só viaja em HTTPS) e sameSite: 'strict' (protege contra ataques de CSRF).
</ul>
</details>

<details>
<summary>HASH CRIPTOGRÁFICA - bcrypt (salvar senha criptografada no banco de dados)</summary>

O bcrypt é um algoritmo e biblioteca de criptografia focado em segurança. Ele é usado principalmente para transformar senhas de usuários em sequências de caracteres seguras (hashes), tornando-as ilegíveis caso um banco de dados seja invadido.
<br>
Para garantir essa proteção, o bcrypt trabalha com três pilares fundamentais:
<ul>
<li>
Mão única: Uma vez gerado o hash de uma senha, não é possível reverter o processo para descobrir qual era a senha original.
</li>
<li>
Uso de "Salt": Ele adiciona caracteres aleatórios à senha antes de criar o hash. Isso garante que, mesmo que dois usuários tenham a mesma senha, o resultado final no banco de dados seja diferente. Isso neutraliza ataques pré-computados, como as rainbow tables.
</li>
<li>
Lentidão proposital: O algoritmo foi desenhado para exigir um alto custo computacional. Isso significa que ele é "lento" de propósito para dificultar e atrasar ataques de força bruta, onde hackers tentam adivinhar a senha testando milhares de combinações por segundo.
</li>
</ul>
</details>

<details>
<summary>NODEMAILER (usado para Redefinição de Senha)</summary>
O Nodemailer é a biblioteca mais popular e utilizada no ambiente Node.js para o envio de e-mails de forma simples, direta e segura. Ele atua como uma ponte entre a sua aplicação (backend) e o provedor de e-mail. Você escreve o código no Node.js e o Nodemailer se encarrega de empacotar e enviar a mensagem.
<br>
Para que o envio realmente aconteça, o Nodemailer precisa se conectar a um servidor de e-mail (como o do Gmail, Outlook, HostGator, etc.) ou a um serviço especializado em disparo de e-mails transacionais (como Mailtrap, SendGrid ou Mailgun).
<br>
O processo básico exige a configuração de duas partes no seu código:
<ul>
<li>O Transporte (Transporter): Onde você define qual servidor de e-mail será usado e as credenciais (usuário e senha ou chave de API) para autenticação.
</li>
<li>
A Mensagem (Mail Options): Onde você define os dados do e-mail em si: remetente, destinatário, assunto, corpo do e-mail (HTML) e anexos.
</li>
</ul>
</details>

<details>
<summary>RESEND E BREVO</summary>
O Resend é uma plataforma e API de e-mail voltada para desenvolvedores e criadores de aplicações, ela serve para automatizar o envio de e-mails transacionais (como recuperação de senha, confirmação de cadastro e notificações) e e-mails de marketing.
<br>
O Brevo é uma plataforma multicanal completa de automação de marketing e vendas voltada para empresas e negócios em crescimento. Ela centraliza a comunicação com clientes em ferramentas como e-mail marketing, SMS, WhatsApp, chat ao vivo e CRM.
<ul>
<li>1. Resend (Notificações do Sistema: E-mails Transacionais)</li>
O Resend está configurado no projeto para cuidar dos disparos automáticos de e-mails transacionais (como confirmações imediatas de compras, avisos de atualização de status de pedidos no banco de dados e interações diretas via API).
<br>
Uso principal: Notificações rápidas e automações, como avisos de status de pedidos ou envios acionados diretamente pelo código da aplicação.
<br>
Observação técnica: Como o projeto roda sob o domínio gratuito da Vercel (.vercel.app), o envio pelo Resend foi configurado utilizando o remetente padrão de testes deles (onboarding@resend.dev), já que domínios compartilhados da Vercel não permitem a verificação completa de DNS (registros SPF/DKIM) exigida para usar um e-mail com domínio próprio.
</li>
<li>2. Brevo (E-mails de Marketing e Leads)</li>
O Brevo é utilizado para a estratégia de crescimento através do formulário do Blindagem Digital & Captação.
<br>
Brevo é a plataforma responsável por armazenar os contatos que entram pelos formulários de captura e por disparar as automações de marketing, newsletters e e-books.
<li>Onde ficam guardadas as chaves?</li>
Ambos os serviços funcionam por meio de chaves de API. Para que o código do seu site funcione com segurança sem expor essas senhas, as credenciais do Resend e do Brevo ficam salvas estritamente como Variáveis de Ambiente (Environment Variables) dentro do painel do seu projeto na Vercel.
</ul>
</details>
